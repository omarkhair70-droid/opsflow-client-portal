# OpsFlow Demo Scenario

## Implemented demo path
1. Client user submits request in portal.
2. Internal user triages and executes via tasks.
3. Internal owner/admin/manager creates quote draft and publishes.
4. Portal client user reviews published quote and submits approval/rejection/changes request.
5. Activity trail records request/task/quote transitions.
